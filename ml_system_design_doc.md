# ML System Design Doc: Система автоматизированного скоринга IT-компетенций (MVP)
## 1. Цели и предпосылки
### 1.1. Зачем идем в разработку продукта?

**Бизнес-цель**:
Сократить расходы на подбор персонала и время закрытия вакансий за счет исключения этапа ручного скрининга резюме и отсева нерелевантных кандидатов.

**Целевой эффект:** Переход от оценки текста резюме к оценке реальных навыков, снижение риска ошибок в найме и стоимости проведения интервью экспертами.

**Почему станет лучше, чем сейчас, от использования ML:**
- **Текущее состояние:** HR тратит в среднем 30-40 минут на первичный анализ одной вакансии и поиск ключевых слов в сотнях резюме. Технический эксперт тратит от 60 до 90 минут на одно интервью, часть из которых быстро завершается из-за несоответствия кандидата базовым hard-skills.
- **С использованием ML:** 
  1. Автоматическая декомпозиция вакансии на атомарные навыки занимает < 1 минуты (вместо полного ручного составления списка компетенций экспертом) + 10 минут валидация экспертом.
  2. Ранжирование кандидатов происходит мгновенно на основе объективных данных из LMS.
  3. HR отбирает топ-N кандидатов с подтвержденным соответствием навыков более 80%. Это позволяет сократить время, затраченное на интервью, проведенное в пустую.

**Что будем считать успехом итерации (Бизнес-метрики):**
1. Доля кандидатов из топ-N рейтинга системы, успешно прошедших первое техническое интервью, составляет не менее 70%.
2. По пилотному набору вакансий система корректно выделяет все обязательные компетенции в 9 из 10 вакансий. Если ниже — пилот считается провальным.
3. Подготовка профиля вакансии с правками занимает < 20 минут (вместо ручного 60–120), а первичный черновик формируется за минуты.

### 1.2. Бизнес-требования и ограничения
**Бизнес-требования:**
- Система должна принимать на вход неструктурированный текст вакансии и выдавать структурированный иерархический список hard-skills.
- Инструментарий для эксперта: возможность за 1-3 клика отредактировать предложенный системой перечень компетенций.
- Автоматический маппинг извлеченных навыков на существующую корпоративную базу знаний (справочник компетенций).
- Выдача HR-менеджеру финального рейтинга кандидатов с прозрачной системой оценивания.


**Бизнес-ограничения:**
- В MVP система ориентирована на ограниченный набор ролей (Data Scientist / Backend) и не претендует на универсальное покрытие всех ИТ-профилей без дополнительной настройки онтологии.
- Фокус только на hard skills. soft skills, локация и зарплатные ожидания в данной итерации не учитываются.
- В качестве источника данных о кандидате не используется текст резюме, так как он содержит шум, неоднозначные формулировки и не подтверждает фактический уровень владения навыками. Входом для оценки служат только проверяемые артефакты из LMS/контест-сервиса: код, результаты тестов, выполненные задания, история попыток и связанные с ними метаданные.
- Конфиденциальность: Данные кандидатов передаются в систему в обезличенном виде (только ID и вектор навыков).

**Что мы ожидаем от конкретной итерации:**
Пилотный запуск на 2-х типах вакансий (Backend, Data Scientist). Результатом является список кандидатов на каждую вакансию, переданный HR-менеджеру без предварительного просмотра резюме.
### 1.3. Что входит в скоуп проекта/итерации, что не входит
**Входит:**
- Модуль извлечения компетенций из текста вакансии.
- Интерфейс правки графа экспертом.
- Двухступенчатая модель ранжирования:
  1. Фильтр по критичным компетенциям.
  2. Ранжирование по близости векторов навыков.
- Интеграционный слой для получения векторов навыков из внешней LMS.

**Не входит:**
- Разработка собственной LMS (используем внешнюю систему).
- Оценка soft skills.
- Автоматическое обновление справочника компетенций в БД (только ручное добавление экспертом).

**Технический долг:**
- Используемую LLM и промпты подбираем в процессе, сравнивая результаты. 
- Используем внешний LLM провайдер, в перспективе использовать self-hosted провайдер
- Оптимизация промптов, батчевая обработка для ускорения работы пайплайна на одной вакансии

### 1.4. Предпосылки решения
- Приоритет у объективных данных из LMS: код, результаты тестов, выполненные задания, число попыток и другие артефакты. Резюме не используются как шумный и недостоверный источник оценки hard skills.
- LLM используется только для подготовки черновика графа компетенций; финальная версия всегда подтверждается экспертом.
- Есть фиксированная онтология компетенций в БД: категории, компетенции, подкомпетенции и синонимы. Извлеченные из вакансии навыки маппятся на нее; новые сущности добавляются только после согласования с экспертом.
- Гранулярность модели вакансии - иерархический граф: категория → компетенция → подкомпетенция.
- Ранжирование кандидатов обновляется при поступлении новых данных из LMS или изменении профиля вакансии.

В MVP система работает только с hard skills и только для ролей Backend / Data Scientist.
